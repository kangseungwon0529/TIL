파일을 공유하는 과정

add(변경사항 추가) → commit(수정사항 값 고정) → push(업로드하는 과정)

#push/pull&clone

push - 내 파일을 업로드 하는것

clone - 최초 일회에 한정하여 모든 파일을 복제하는것

pull - 처음 모든 파일을 복제했던 것을 다 가져올 필요가 없기 때문에 추가된 항목만

   가져올 때 수행하는 명령어

push 하기 전엔 항상 pull을 해서 가져 온 후에 작업이 진행되어야

git 상에서 파일의 오류가 생기지 않는다.

# git ignore

.gitignore 파일을 생성하면
그 무시할 파일을 입력할 수 있는 ignore창이 생성된다.
이 창 내부에 파일명.파일형식을 작성하여 저장하여 주면
git 에 업로드 되지 않도록 설정할 수 있다.
다만, 이미 업로드 되어있는 파일은 이그노어 파일로 지정할 수 없다.
